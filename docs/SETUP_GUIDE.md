# KAISHA AI オーケストレーター セットアップガイド

## 1. 前提

- Node.js 20 以上
- pnpm

## 2. インストール

```bash
pnpm install
```

## 3. 起動

```bash
pnpm dev
```

起動後:

- サーバー: `http://localhost:3200`
- UI: `http://localhost:3201`

## 4. 初回セットアップウィザード

初回表示時にウィザードが開きます。

1. 会社名とプレフィックスを入力
2. サーバー表示名とローカルトークンを設定
3. 利用スキルを確認して完了

完了後、設定内容は SQLite に保存されます。

## 5. API キー設定の考え方

初期版では認証を簡易化しています。実運用で外部 API キーを持たせる場合は:

1. `設定` 画面を開く
2. `インスタンス設定` のトークン欄や運用メモ欄を使って管理方針を記録
3. 必要に応じて `server/src/routes/settings.ts` と `settings` テーブルに項目を追加

現状はローカル単一ユーザー運用向けです。

## 6. エージェント追加手順

1. `エージェント` を開く
2. `エージェント追加` を押す
3. 名前、役割、アダプター、ステータス、スキル、指示文を入力
4. `保存する` を押す

対応アダプター例:

- OpenClaw
- Claude Code
- Codex
- HTTP
- Bash

## 7. ルーティン設定手順

1. `ルーティン` を開く
2. `ルーティン追加` を押す
3. 実行エージェントを選ぶ
4. `scheduleType` に `cron` または `interval` を設定
5. cron 式またはインターバル分数を入力
6. 保存する

手動実行したい場合:

1. 対象ルーティンカードの `実行` を押す
2. 実行ログが更新されることを確認

## 8. ダッシュボードの見方

- サマリーカード: 稼働エージェント、進行中タスク、今月コスト、承認待ち
- アクティビティ: 直近の操作ログ
- リアルタイム稼働カード: SSE で Heartbeat を反映

## 9. DB の保存先

SQLite ファイル:

```text
server/data/kaisha-ai.sqlite
```

削除すると seed 付きで再初期化されます。
